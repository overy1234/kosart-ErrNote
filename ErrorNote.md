# ErrorNote

> AI 작업 중 발생한 에러 및 문제점을 기록하고, 재발 방지와 최종 검토에 활용하는 참고 노트입니다.

---

## Quick Checklist

**작업 시작 전**
- [ ] 관련 ErrorNote 항목 읽기
- [ ] 유사한 에러 패턴 있는지 확인
- [ ] 재발 방지 항목 먼저 적용

**작업 중**
- [ ] 에러 발생 시 즉시 기록 또는 기존 항목에 재발 메모 추가

**작업 완료 전**
- [ ] ErrorNote 재확인
- [ ] 기록된 실수가 반복되지 않았는지 체크
- [ ] 새로운 패턴 발견 시 추가

---

## 기록 형식

```
### [날짜] 제목
- **상황**: 어떤 작업 중 발생했는지
- **문제**: 구체적으로 어떤 에러/문제였는지
- **원인**: 왜 발생했는지 (파악된 경우)
- **시도했지만 실패한 방법**: 효과 없었던 접근법 (다음에 같은 시도 반복 방지)
- **해결**: 어떻게 해결했는지
- **검증**: 어떤 결과/화면/명령으로 해결됐는지 확인했는지
- **재사용 조건**: 어떤 상황에서 이 기록을 다시 참고해야 하는지
- **재발 방지**: 다음에 같은 실수를 하지 않으려면
- **태그**: #카테고리
```

---

## 기록 기준

### 기록해야 하는 것
- 반복될 가능성이 큰 문제
- 원인 찾는 데 오래 걸린 문제
- 콘솔/UI가 헷갈리는 문제
- 빌드/배포/검증 절차 관련 문제
- 프로젝트별 특수 규칙

### 기록하지 않아도 되는 것
- 단순 오타
- 일시적 네트워크 오류
- 한 번만 발생한 외부 서비스 장애
- 재발 가능성이 낮은 일회성 문제
- 임시 진행 상황 (작업 요약은 별도 관리)

## 보안 규칙

절대 기록하지 않는 것:
- API 키, OAuth 토큰, 세션 ID
- .env 파일 값
- 토큰이 포함된 URL
- 비밀번호, 인증 정보
- 프로덕션 서버 주소

**Bad:** `- **문제**: API_KEY=sk-xxxx 로 인증 실패`
**Good:** `- **문제**: OpenAI API key 환경변수 누락으로 인증 실패`
